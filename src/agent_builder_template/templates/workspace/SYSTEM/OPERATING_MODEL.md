# Operating Model

This workspace is designed to run like a disciplined delivery loop rather than a loose collection of prompts.

## Primary Flow

`Brief -> Clarify -> Plan -> Architect -> Build -> Critic -> Regression -> Report -> Next Action`

## How To Use It

1. Start from `PROJECT/PROJECT_BRIEF.md`.
2. Confirm project context and architecture direction.
3. Break work into reviewable tasks.
4. Move one task into active work.
5. Execute the smallest meaningful slice.
6. Review, validate, and report.
7. Update state and make the next action explicit.

## Fast Path

Use a compressed flow for small, low-risk work with stable context:

`Builder -> focused validation -> Report -> Artifact Steward`

## Full Path

Use the full path for new projects, unclear requirements, architecture work, or medium and larger changes:

`Intake -> Planner -> Architect -> Builder -> Critic -> Regression -> Report -> Artifact Steward`

## Design Principle

The system should add just enough structure to improve delivery quality, handoff quality, and decision quality. If a step is not improving one of those, simplify it.
