# Quality Bar

This file defines the default standard for work in this workspace.

## Required Characteristics

- Clear scope
- Reviewable changes
- Explicit assumptions
- Practical validation
- Useful reporting
- Controlled scope growth

## Delivery Standards

- Prefer minimal, understandable changes over cleverness.
- Keep plans and architecture proportional to the project.
- Avoid unnecessary refactors during delivery work.
- Make approval-sensitive decisions visible before implementing them.
- Record important risks and tradeoffs instead of hiding them.

## Validation Standards

- Use targeted checks that match the task risk.
- Increase validation depth when shared behavior or regression risk grows.
- State confidence level honestly when coverage is partial.

## Documentation Standards

- Keep the brief and context current enough for handoff.
- Record decisions that materially affect delivery.
- Leave reports short, factual, and useful for the next decision.
