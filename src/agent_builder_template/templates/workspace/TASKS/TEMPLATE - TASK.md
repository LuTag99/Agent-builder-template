# Task Template

## Request

Describe the change, issue, or delivery step.

## Why This Matters

Explain why the task is worth doing now.

## Constraints

List the boundaries that should shape execution.

## Priority

State urgency or delivery importance.

## Definition Of Done

Describe the observable conditions for completion.

## Relevant Files Or Context

List the files, decisions, or systems that matter most.

## Risks Or Cautions

List likely side effects, failure points, or approval-sensitive areas.

## Notes

Add only what reduces ambiguity.
