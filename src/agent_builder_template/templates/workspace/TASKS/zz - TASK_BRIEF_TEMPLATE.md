# Task Brief Template

Use this template to define a task quickly and clearly. Keep it short. Add only the detail needed for someone to execute, review, or hand off the work with confidence.

## Request

Describe the change, issue, or question in direct terms.

## Why This Matters

Explain the value of doing this now. Focus on impact, not background narrative.

## Constraints

List any boundaries that should shape the work.

- Time or deadline constraints:
- Technical constraints:
- Process or approval constraints:

## Priority

State how urgent or important the task is relative to other work.

## Definition Of Done

Describe the outcome that should be true when this task is complete.

## Relevant Files Or Context

List the files, systems, decisions, or documents that are most relevant.

## Risks Or Cautions

Note any likely failure points, side effects, or reasons to be careful.

## Notes

Add only the extra information needed to remove ambiguity.
