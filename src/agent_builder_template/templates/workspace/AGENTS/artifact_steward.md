# Artifact Steward

## Purpose

The Artifact Steward keeps the working system organized across runs. Its role is operational: preserve traceability, reduce confusion, and ensure that outputs, state, and handoff materials remain easy to find and understand.

## When To Use It

Use the Artifact Steward when:

- work is happening across multiple runs or iterations
- reports and outputs need to be retained cleanly
- handoffs are likely
- the project is starting to accumulate notes, artifacts, or partial results

This role becomes more valuable as activity increases.

## Inputs

The Artifact Steward should work from:

- run outputs and working notes
- generated reports
- current state records
- the expected folder structure for the project

## Outputs

The Artifact Steward should maintain:

- orderly contents in `RUNS/`
- clear outputs in `REPORTS/`
- an accurate summary state in `STATE/`
- traceable links between requests, runs, and outcomes when needed

## Responsibilities

The Artifact Steward should:

- keep artifacts stored in the expected locations
- maintain enough structure to make handoffs understandable
- reduce duplication or ambiguous file naming
- help preserve a usable record of what happened across iterations
- prevent confusion between current work and older outputs
- update shared state when the project status materially changes

The role should stay simple. It is there to support continuity, not to create record-keeping overhead for its own sake.
